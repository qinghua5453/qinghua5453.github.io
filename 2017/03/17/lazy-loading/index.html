lazy-loading

发表于 2017-03-17 | 分类于 plugin |

图片懒加载应用很广，因为在大访问量的网站，如果初次打开页面就加载所有的图片，势必要求很高的带宽和服务器的性能要求，而且用户在浏览页面上部分的时候就已经把下部分的图片加载出来了，本来就是一种资源的浪费，所有懒加载插件还是很有必要的，下面就用原生的js写一个比较容易理解的懒加载。我们先屡屡思路。

思路

1、用户在视窗内才加载图片，视窗外的图片不加载

2、如何用js判断元素是否在视窗内，用ele.getBoundingClientRect()方法

3、已经加载了的图片不在重新替换src，需用一个变量标记每一个元素

源码说明

其实写这个插件的时候拆分成了几大区块

1、构造函数主要构造一个对象，并在里面初始化变量

2、_detectElementIfInScreen 函数是原型链的一个函数，主要是循环元素判断是否在可视区域内。这里判断可视区域条件比较多，容易漏考虑，而且用到了getBoundingClientRect()方法，摈弃了原生的offsetTop 和offsetLeft方法，理解起来更直观，而且新的getBoundingClientRect()直接返回的6大变量，也便于我们调试。
